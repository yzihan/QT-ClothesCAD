这有两个文件夹：1.release 2.yzh3551

其中release文件夹中是release版本的执行文件
您直接双击点开可能会出现系统错误提示：找不到Qt5Svg.dll
这个报错提示代表缺乏运行该exe的环境，您有以下两种解决办法：
1.用命令行把环境变量提供给exe（程序运行需要dll动态链接库，
直接将程序安装目录X:\Qt5.0.2\bin目录加入到Path环境变量中去）
2.找到您电脑中保存Qt5Svg.dll的文件夹，把exe复制进那个文件夹，此时就可以运行

我提供了我的Qt5Svg.dll，如果是应用程序无法正常启动0xc000007b的情况
点击电脑左下角的开始菜单，在搜索栏输入cmd.exe，按回车键打开管理员命令窗口
在管理员界面输入sfc /scannow，然后按Enter键，扫描之后应该就好了

yzh3551文件夹中是程序的源代码
您也可以直接通过该文件夹中的“yzh3551.pro”在Qt5中运行程序
倘若不能编译，可能需要设置下build的生成地址（配试您的电脑目录）
不如直接把我的程序复制您电脑生成build的位置

这里需要注意的是：如果您用的是VS，这里的后缀是.sln，.pro是QT Creator的后缀
icons中是一些为了美观而使用的图标，这些图标可以从网上下载到
.h .cpp .ui文件是要求文件 .qss是用于界面渲染的文件

您可以先重新构建所有文件，再编译运行
